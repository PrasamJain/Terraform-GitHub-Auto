# Terraform-GitHub-Auto

> Beginner-friendly Terraform project to create AWS EC2 instances via GitHub Actions CI/CD.

---

## 📁 Project Structure

```
Terraform-GitHub-Auto/
├── main.tf                    ← Root entry point (provider + module call)
├── variables.tf               ← Variable declarations
├── terraform.tfvars.json      ← ✏️  EDIT THIS to change EC2 settings
├── modules/
│   └── ec2/
│       ├── main.tf            ← EC2 resource definition
│       ├── variables.tf       ← Module input variables
│       └── outputs.tf         ← Module outputs (instance_id, public_ip, etc.)
├── tfstates/                  ← State files saved here (one per EC2 name)
│   └── <ec2_name>.tfstate     ← Auto-generated by CI/CD pipeline
└── .github/
    └── workflows/
        └── terraform.yml      ← CI/CD pipeline
```

---

## ✏️ How to Create or Update an EC2

**All you need to change is `terraform.tfvars.json`:**

```json
{
  "aws_region":     "us-east-1",
  "ec2_name":       "my-first-ec2",
  "ami":            "ami-0c02fb55956c7d316",
  "instance_type":  "t3.micro",
  "key_name":       "",
  "tags": {
    "Environment": "dev"
  }
}
```

---

## 🔄 Name-Based State Logic

| What you do | What happens |
|---|---|
| Change `ec2_name` to a **new name** | New state file created → **New EC2 launched** |
| Keep `ec2_name` the **same** | Same state file used → **Existing EC2 updated** |

State files are stored in `tfstates/<ec2_name>.tfstate` and committed back to this repo automatically after each pipeline run.

---

## 🔑 Required GitHub Secrets

Go to: **Repo → Settings → Secrets and variables → Actions**

| Secret name | Value |
|---|---|
| `AWS_ACCESS_KEY_ID` | Your AWS access key |
| `AWS_SECRET_ACCESS_KEY` | Your AWS secret key |

---

## 🚀 Running Locally

```bash
# 1. Install Terraform: https://developer.hashicorp.com/terraform/downloads
# 2. Set AWS credentials
export AWS_ACCESS_KEY_ID="..."
export AWS_SECRET_ACCESS_KEY="..."

# 3. Init
terraform init

# 4. Plan (replace 'my-first-ec2' with your ec2_name)
terraform plan -var-file=terraform.tfvars.json -state=tfstates/my-first-ec2.tfstate

# 5. Apply
terraform apply -var-file=terraform.tfvars.json -state=tfstates/my-first-ec2.tfstate
```
